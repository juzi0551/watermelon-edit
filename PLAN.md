# 小说校稿工具开发计划

## 项目概述
开发一个基于大模型的小说校稿工具，支持DeepSeek、Kimi等国内常用大模型，提供Web界面进行文档上传、校对、结果查看和修改。

## 开发周期
预计总工期：15个工作日（含打包分发）

## 分发方案（重要）
**目标用户**：普通作者/编辑，不懂技术，必须双击即用
**目标平台**：Windows + macOS 双平台
**核心思路**：开发用 Python，分发用 PyInstaller 打包成的独立程序（内含 Python 解释器 + 依赖，用户无需安装任何环境）

### 打包方式
- 用 **PyInstaller** 将 FastAPI 后端 + 打包后的 React 静态文件，连同 Python 解释器一起打成单个可执行文件
- 程序启动时自动在本地起服务、自动打开浏览器，用户无感知
- PyInstaller **不能跨平台编译**：
  - 在 Mac 上打包 → 产出 `.app`
  - 在 Windows 上打包 → 产出 `.exe`
  - 开发者有 Mac + Windows 两台机器，各打各的即可
- （可选）后续可配置 GitHub Actions，推送代码后自动双平台构建并发布安装包

### 用户使用流程
```
下载 → 解压 → 双击"小说校稿工具" → 浏览器自动打开 → 使用
```
全程无需命令行、无需安装 Python、无需配置环境。

## 技术实现细节

### 1. 文档处理流程
```
上传docx → 解析文档结构 → 提取章节内容 → 分段处理 → 调用大模型 → 合并结果 → 生成报告
```

### 2. 大模型调用策略
- **分段处理**：按章节分段，每段控制在3000-4000 tokens
- **并行调用**：使用asyncio并行处理多个章节
- **重试机制**：对失败请求进行自动重试（最多3次）
- **结果缓存**：避免重复检查相同内容

### 3. 结果数据结构
```json
{
  "document_id": "doc_123",
  "model_used": "deepseek-chat",
  "chapters": [
    {
      "chapter_id": "ch_1",
      "title": "第一章",
      "errors": [
        {
          "type": "typo",
          "paragraph_index": 2,
          "original_text": "原文内容",
          "suggested_text": "修改建议",
          "severity": "high",
          "description": "错别字：'的'应为'地'"
        }
      ]
    }
  ]
}
```

### 4. 前端界面设计
- **主界面**：文件上传区域 + 模型选择 + 开始校对按钮
- **结果界面**：错误列表（按类型分组）+ 修改建议 + 应用修改按钮
- **导出界面**：预览修改效果 + 导出docx按钮

## 风险评估与应对

### 1. 大模型响应不稳定
- **风险**：API调用失败或返回格式错误
- **应对**：实现重试机制，提供默认响应格式解析

### 2. docx格式保留复杂
- **风险**：某些格式信息丢失
- **应对**：使用python-docx的高级功能，测试多种文档格式

### 3. 长文本处理性能
- **风险**：处理大文档时内存不足
- **应对**：流式处理，分批加载，优化内存使用

### 4. 中英文混合处理
- **风险**：语言检测不准确
- **应对**：使用jieba进行中文分词，结合语言检测库
